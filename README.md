# Team Project Repository

## Workflow Followed

1. Created a GitHub repository.
2. Cloned the repository to local machine.
3. Created separate branches for each module:
   - login-module
   - database-module
   - ui-module
4. Made changes in respective branches.
5. Committed and pushed changes to GitHub.
6. Created Pull Requests for each branch.
7. Reviewed and merged branches into main.
8. Resolved merge conflicts manually when required.

## Branch Strategy

Each module was developed in a separate branch.
Only reviewed and approved code was merged into the main branch.
