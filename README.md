# Team Project Version Control

## Project Overview
This repository demonstrates the implementation of version control using Git and GitHub for a team-based project.

## Workflow Followed

1. Created a GitHub repository.
2. Cloned the repository to the local machine.
3. Created separate branches for each module:
   - login-module
   - database-module
   - ui-module
4. Developed features independently in their respective branches.
5. Committed and pushed changes to GitHub.
6. Created Pull Requests for code review.
7. Merged approved branches into the main branch.
8. Resolved merge conflicts manually during branch integration.

## Branch Strategy

- The `main` branch contains stable and reviewed code.
- Feature branches were used for module development.
- Pull Requests ensured controlled merging and collaboration.
- Conflicts were resolved before final integration.

## Conclusion

The repository successfully demonstrates branch creation, collaboration, pull request workflow, and conflict resolution using GitHub.
